- (decision) Slug: kagamios (inherited from prd-KagamiOS-2026-07-02 / architecture-KagamiOS-2026-07-02 folders).
- (note) Why: a vision to realize, with a mandate-adjacent forcing function. A single researcher (KagamiOS's own first user, per docs-spec's dogfooding framing) needs a way to turn a vague research intuition into evidence-backed candidate directions without either (a) doing the full landscape survey by hand or (b) letting an AI's fluent framing quietly become the researcher's own framing (E6 homogenization). KagamiOS exists to do the tractable landscape work automatically while keeping every consequential judgment mechanically reserved for the human, and to prove that discipline in code (write-guards, generation windows, versioned provenance) rather than prompt convention, because the runtime shape decision (spine AD-1/AD-2) was made explicitly to make that provable.
- (capability) CAP-1 intent: A researcher can drive one investigation through the deterministic state machine (Frame->Map->Deepen->Synthesize->Locate) with loop-backs and skip-waivers, reaching Dissolved or Dormant, or an accepted Gap Register (Decided/Propose deferred to v2, non-precluded). success: Every state transition is a logged state_transition event with a cause annotation on loop-backs; derived per-cluster state (spine AD-20 pure function) matches the schema-registry exit criteria with no manual override. PRD FR-1..7; spine AD-9, AD-14, AD-20, AD-24.
- (capability) CAP-2 intent: The system stores every artifact as an immutable-versioned, schema-validated, provenance-tracked file so no AI write can reach human-only fields and no human edit can be silently lost. success: A write-guard violation is mechanically refused and logged (zero occurrences reach accepted state in the AD-25 golden fixture); a human edit to current.md always survives a concurrent queued AI write (AD-16 scan-before-write, refuse-and-proposed-diff). PRD FR-9..16, FR-30..35; spine AD-2, AD-3, AD-6, AD-15, AD-16, AD-19.
- (capability) CAP-3 intent: The system asks the researcher only the questions a schema/artifact diff could not resolve itself, batched at most 5 at a time, ranked by leverage, always against evidence already shown. success: Every Question Ledger entry other than the two E6 unprimed questions carries a non-empty target and leverage class; no ASK batch in the event log exceeds 5; a rent audit shows consumed_by populated for every non-deferred question. PRD FR-17..24; spine AD-8, AD-17.
- (capability) CAP-4 intent: Role-scoped AI work (Scout, Cartographer, Historian, Skeptic, drafting worker) runs in isolated, read-set-bound contexts so no role can act outside its charter or see another role's drafting rationale. success: Every llm_call event carries a non-null role tag and a valid, unexpired engagement token (spine AD-4); a charter-violation audit (Skeptic proposing, Historian speculating, non-Scout touching raw corpus) returns zero matches on the golden fixture. PRD FR-25..29; spine AD-4, AD-12.
- (capability) CAP-5 intent: No direction-shaped content (Candidate Direction or later) can be generated before the run's Gap Register is accepted, and no AI framing artifact can be created before the corresponding unprimed researcher answer is recorded. success: The schema registry's window table refuses artifact creation outside its window (verified by the AD-25 fixture); Frame/Propose artifact creation timestamps are always later than their unprimed ledger entries; violations land in premature_ideas/, never silently dropped. PRD FR-24, FR-46; spine AD-9.
- (capability) CAP-6 intent: A researcher's literature search (Scout) draws from multiple interchangeable providers (OpenAlex, Semantic Scholar, arXiv, GitHub) with no provider hard-coded at any call site, producing reusable Markdown paper cards. success: Swapping the configured default provider (config.yaml) requires no code change and passes the same adapter contract test; a paper card is computed once per paper and reused across runs (corpus/ cache hit, no recompute). PRD FR-25 (corpus scope); spine AD-7, AD-18.
- (capability) CAP-7 intent: The system logs enough about its own run, deterministically, to let a human design owner later evaluate KagamiOS's effectiveness, without any runtime decision ever reading that log back. success: Deleting a completed run's event log changes no other run's behavior (AD-25 invariance test); kagami metrics recomputes identical numbers from the same log; the ledger's question-class key and run-id outcome-join fields are populated from run 1 even though the cross-run audit loop itself is v2. PRD FR-36, FR-37, FR-39, FR-47 (non-preclusion); spine AD-11, AD-14.
- (capability) CAP-8 intent: A researcher installs KagamiOS as a Claude Code plugin alongside BMAD and runs it entirely on their own machine, with all researcher-owned data living in their project, never inside the plugin. success: Plugin install/uninstall never touches _kagami-output/; uv run --project resolves kagami from the plugin regardless of the invoking cwd; an upgraded schema registry refuses to mutate a run written under an older registry rather than corrupting it. PRD Platform NFR (section 5); spine AD-1, AD-5, AD-13, AD-23.
- (constraint) All mechanical guarantees (write-guards, generation windows, ASK batching, versioning, event logging, staleness) must be deterministic Python code inside the plugin core, never prompt convention or SKILL.md instruction — the reason the runtime is a gatekeeper-kernel plugin, not a thin skill (spine AD-1, AD-2).
- (constraint) KagamiOS ships as a Claude Code plugin co-installable with BMAD, following its layout conventions (skills+scripts+hooks, uv toolchain, memlog.py-style entrypoint contract) — not a standalone application (spine AD-1, AD-23; PRD Platform NFR).
- (constraint) The deterministic core (kagami/) must remain standalone-capable as a library importing no harness API, so a future non-plugin runtime is an adapter, never a rewrite (spine AD-5).
- (constraint) Literature-provider choice is never hard-coded at a call site; OpenAlex and Semantic Scholar are peer adapters behind one port, default is config.yaml (spine AD-7) — provider policy/availability drift (OpenAlex's Feb-2026 mandatory-key change) is the standing proof this constraint bites.
- (constraint) v1 scope is Frame through Locate only, terminating at an accepted Gap Register (or Dissolved/Dormant); Propose and Decide are v2 — but the schema registry, state enum, and window table ship complete from v1 so v2 is additive, never a breaking migration (PRD section 7.1; spine AD-14).
- (constraint) Privacy default is off: no run data leaves the researcher's local store without explicit opt-in, and any future shared aggregation is content-stripped (event shapes/counts/classes only) (PRD FR-39).
- (constraint) Researcher-owned data (runs, corpus cache, entity registry, researcher profile) lives under a configurable output root in the user's project, never inside the plugin install (spine AD-13).
- (non-goal) Anthropomorphic AI personas, a portfolio terminal event, a first-class knowledge hierarchy, a database as ground truth, RAG/vector index over the artifact graph, mechanical Decide-gate closure, analytics-driven self-modification, or any lifecycle stage beyond the bounded micro-probe — all standing refusals carried unchanged from docs-spec/ (PRD section 6, 10 items).
- (non-goal) Multi-researcher or lab-as-shared-gate use in v1; an advisor consumes the finished handoff bundle but never gates a run from inside it (PRD section 2.2).
- (non-goal) A local embedding index over paper cards in v1 — a recorded, researcher-approved deviation from docs-spec's corpus-tier design; the interim substitute is citation-graph communities plus provider-side relevance search (spine Deferred).
- (non-goal) A CLI or non-plugin runtime, learned/adaptive dispatch routing, machine-side token budgets, and the cross-run analytics / Design Audit Report loop — all deferred past v1 with their own adoption triggers (PRD section 7.2; spine Deferred, AD-14's run-1 schema locks aside).
- (decision) Success signal: a researcher runs one real investigation through KagamiOS v1 from Intuition Note to an accepted Gap Register or a Dissolution Memo, using only the minimal-run profile, with (a) zero write-guard or generation-window violations in the run's event log, (b) every consumed question showing consumed_by traceability, and (c) the run surviving a Claude Code session restart mid-run without state corruption (AD-15/AD-25's crash-consistency guarantee exercised for real, not just in the golden fixture). Demonstrable via kagami metrics output plus a manual read of events.jsonl. (PRD section 8 interim metric + spine AD-25.)
- (assumption) MVP scope breadth (full artifact chain through an accepted Gap Register, not narrower) and the skip-hand-run-validation decision (docs-spec OQ1) are carried from the PRD as settled, not re-opened here (PRD section 9 OQ-A, resolved).
